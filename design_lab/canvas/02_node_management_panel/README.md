# 02 Node Management Panel

This study isolates the right-side node management panel that appears when a canvas node is selected.

Scope:

- panel shell and overall footprint
- relationship between selected node and side panel
- header density and metadata treatment
- slider, select, checkbox, and value-row styling
- footer action block

Reference goals:

- match the narrow, dark, utility-first control panel from the reference
- keep the controls dense without feeling cramped
- make the panel feel like part of the current Stitchly canvas language
- preserve enough structure for future node-specific controls

Current choices:

- static selected `Send Email` node on the left for context
- open management panel on the right
- mixed control set: sliders, select, toggles, footer action row
- lava-accented shell with restrained white line work

Things to judge:

- is the panel too tall or too wide?
- should the header feel more technical or more product-like?
- are the control rows dense enough for repeated node editing?
- should the footer CTA be stronger or quieter?
