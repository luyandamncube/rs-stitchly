Reference Analysis

Dashboard Screen Breakdown

This study captures the structure, density, and interaction language of the supplied dashboard reference before we adapt it into Stitchly’s post-login product shell.

Overall Composition

High-level read
  • Contained dashboard shell floating on a clean neutral background.
  • Minimal left sidebar plus wide right-side data surface.
  • Calm product tone driven by spacing and soft surfaces.
  • Operations-first table view with just enough summary above it.
Primary layers
  • Large rounded outer frame.
  • Narrow vertical navigation sidebar.
  • Main content area with title, filters, metrics, and table.
  • Small embedded notifications and floating utility action.

Shell And Navigation

Sidebar behavior
  • Primary nav sits high in the sidebar with generous spacing.
  • Active item is indicated quietly with stronger text and a side marker.
  • Notification tile is integrated into the sidebar, not separate chrome.
  • Profile and settings live at the bottom as utility destinations.
Main content behavior
  • Compact title area, not hero-like.
  • Filter row sits directly above the data it controls.
  • Export and search are present but visually restrained.
  • Everything is aligned to a clean table-centric grid.

Typography And Density

Type observations
  • Neutral sans with compact product/UI rhythm.
  • Hierarchy relies on weight more than large size jumps.
  • Muted labels and stronger values drive the scan pattern.
  • Status text stays small and functional.
Density observations
  • Overall shell is spacious, but data areas are compact.
  • KPI strip is lightweight instead of becoming a grid of cards.
  • Table rows are dense but legible.
  • Soft separators do most of the organization work.

Key Elements

Filter and KPI language
  • Pill filters for category, scope, and time.
  • `More filters` keeps the top bar from getting crowded.
  • KPI values live in one horizontal summary strip.
  • Small deltas and success indicators provide lightweight context.
Table and utility language
  • Operational table is the main feature of the page.
  • Statuses use compact icon-plus-text treatments.
  • Errors are inline and practical, not dramatic.
  • Floating bottom-right `Actions` pill creates a secondary utility anchor.

Stitchly Adaptation

Preserve
  • Contained outer shell.
  • Calm narrow sidebar.
  • Inline KPI strip above the table.
  • Dense readable table-first layout.
  • Restrained use of status color.
Replace
  • Light neutral palette with our dark `True Black + Lava Core` system.
  • Generic business analytics content with Stitchly workflow/run content.
  • Static alerts with execution-aware problems and agent notifications.
  • Business dashboard language with workflow-editor product language.
