# 03 Dashboard Sidebar

Type: `navigation-and-utility-study`

Purpose:

- focus on the left dashboard rail as a first-class design surface
- refine navigation rhythm, alert placement, utility grouping, and profile/footer treatment
- test whether Stitchly’s contained dashboard can support a calmer, more product-like utility shell

What this sample is testing:

- slightly wider sidebar with stronger section grouping
- softer active navigation treatment than a loud accent-first rail
- embedded alert and utility cards as part of the sidebar system
- lower navigation, workspace utilities, and profile context as one coherent stack
- how much the main panel can fade back while the sidebar becomes the star

What this sample intentionally includes:

- dashboard shell reused from `01` and `02`
- sidebar-focused composition
- primary nav
- alert card
- lower utility list
- profile/footer area
- muted main-panel placeholder so the sidebar can be judged in context

What this sample intentionally does not solve yet:

- final notification logic
- collapsed sidebar behavior
- keyboard/hover interaction rules
- mobile dashboard navigation
- final workspace/account menu patterns

Shared styling:

- this sample uses `../shared.css`
- sidebar-study helpers are appended to the shared stylesheet rather than using a local override file

Review questions:

- does the sidebar feel like a strong product surface on its own?
- is the widened rail better than the narrower shell from `01` and `02`?
- do the alert and utility blocks feel useful or too card-heavy?
- should Stitchly’s contained dashboard lean more into this utility-sidebar pattern?
