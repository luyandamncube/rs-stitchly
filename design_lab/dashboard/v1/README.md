# Dashboard Lab `v1`

This folder contains the first isolated design studies for Stitchly dashboard and post-login application surfaces.

Purpose:

- analyze reference dashboards before adapting them
- refine dashboard shell, navigation, data tables, and utility surfaces in isolation
- approve layout and interaction language before production implementation

Convention:

- `shared.css`
  Shared dashboard-language stylesheet for the current contained-shell design pass.
- `00_xxx/`
  Reference analysis and foundational direction.
- `01_xxx/`
  First dashboard shell studies.
- `02_xxx/`
  Data surface and table studies.
- `03_xxx/`
  Navigation and utility pattern studies.
- `04_xxx/`
  Collapsed shell and compressed navigation studies.
- `05_xxx/`
  Mobile and narrow-layout dashboard studies.
- `06_xxx/`
  Detail panel and contextual inspection studies.
- `07_xxx/`
  Empty and no-results dashboard studies.
- `08_xxx/`
  Loading and refresh-state dashboard studies.
- `09_xxx/`
  Filter and search dashboard studies.
- `10_xxx/`
  Bulk action and selection studies.
- `11_xxx/`
  Notification and alert-center studies.

Preferred progression:

1. reference analysis
2. shell layout study
3. data table and metrics study
4. sidebar and notification study
5. collapsed sidebar and shell-state study
6. mobile and narrow-layout adaptation
7. detail panel and contextual inspection study
8. empty and no-results study
9. loading and refresh-state study
10. filter and search study
11. bulk action and selection study
12. notification and alert-center study

This folder is static-first and should remain separate from the production app shell.
