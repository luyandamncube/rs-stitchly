# 01 Login Shell

Type: `layout-study`

Purpose:

- translate the login reference analysis into a first Stitchly auth shell study
- prove the desktop split, left brand theater, and right rounded form card
- establish a shared login-language CSS foundation for later auth studies

What this sample is testing:

- broad screen composition before form detail work
- balance between left-side brand stage and right-side login card
- card radius, inset, and spacing proportions
- top-level CTA anchoring
- how Stitchly branding fits into the reference shell language

Reference direction:

- the supplied split-screen luxury login reference

What this sample intentionally includes:

- left-side Stitchly wordmark and symbol stage
- subtle guide-line structure
- right-side rounded auth card
- oversized `Log in` title
- placeholder field lines and helper actions
- detached circular CTA

What this sample intentionally does not solve yet:

- final field design
- final helper-copy language
- exact button styling for production
- final left-side brand-panel content
- mobile-first auth behavior

Shared styling:

- this sample uses `../shared.css`
- the stylesheet should become the shared auth-shell foundation for later login studies

Review questions:

- does the overall split feel balanced?
- does the right-side card feel inset and premium enough?
- does the left-side Stitchly stage carry enough identity without getting noisy?
- should the circular CTA survive into the next study?
- does the shell feel like a strong adaptation of the reference rather than a generic app login?
