# 03 Login Brand Panel

Type: `surface-study`

Purpose:

- focus directly on the left side of the login shell
- replace the finance-style monument feeling with a Stitchly-specific workflow brand stage
- test whether the symbol can carry a stronger systems/editor identity before final implementation

What this sample is testing:

- the Stitchly symbol as the central monument
- workflow-oriented supporting motifs around the symbol
- supporting copy beneath the monument
- balance between brand richness and restraint
- whether the right-side form can be visually quiet while the left panel leads

Reference direction:

- the left-side brand theater area from the supplied split-screen login reference

What this sample intentionally includes:

- larger central Stitchly symbol
- subtle orbit/guide treatment around the monument
- floating workflow chips such as `Flows`, `Nodes`, `Agents`, and `Runs`
- short supporting brand copy
- muted placeholder auth panel on the right for context

What this sample intentionally does not solve yet:

- final production copy
- final iconography for brand motifs
- the exact motion language for the left panel
- whether floating chips should stay literal or become more abstract

Shared styling:

- this sample uses `../shared.css`
- the brand-panel classes added here should remain reusable for later auth and marketing studies

Review questions:

- does the left panel now feel more like Stitchly and less like a generic premium app?
- are the floating workflow chips helpful or too literal?
- does the supporting copy add clarity without clutter?
- should this richer brand-panel direction continue into later studies?
