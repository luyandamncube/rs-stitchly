# 00 Canvas Node Menu

This study isolates the real canvas menu button set using the same visual language as the canvas control.

Scope:

- compact control-style dock
- real button set for Stitchly canvas navigation and node families
- click-open output popout
- quick-access card menu

Current choices:

- `Output` is open by default for review
- the dock now includes:
  - logo
  - canvas
  - trigger
  - input
  - compute
  - data movement
  - control
  - output
  - system
  - exit
- the dock intentionally matches the canvas control proportions and radius language

Things to judge:

- does the dock feel close enough to the canvas control?
- do these icons feel distinct enough at this size?
- should the popout be taller, wider, or denser?
- should the active lava tile stay this size?
