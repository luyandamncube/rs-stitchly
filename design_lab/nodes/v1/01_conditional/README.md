# 01 Conditional

Type: `surface-study`

Purpose:

- create the first conditional node sample in the design lab
- follow the supplied conditional reference closely in structure
- keep Stitchly's dark `True Black + Lava Core` palette
- prove that the shared node-language CSS can support grouped decision nodes

What this sample is testing:

- compound header icon treatment
- horizontal key-value row for the condition expression
- grouped stats panel for branch outcomes
- stacked output handles on the right edge
- shared-CSS support for condition-style node structure

Reference direction:

- the `Conditional` node from the provided sample image set

Still unresolved:

- whether branch handles should stay in the same lava family or diverge more strongly
- whether grouped stat panels should be denser for production nodes
- whether conditional headers should show type name only or instance label later

Shared styling:

- this sample uses `../shared.css`
- it should remain inside the shared node-language system unless a later review justifies a local override

Review questions:

- does the grouped panel feel close enough to the sample?
- do the stacked output handles feel balanced?
- is the conditional row hierarchy clear enough?
- does the dark adaptation still feel compact and clean?
