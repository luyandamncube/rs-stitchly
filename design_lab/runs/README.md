# Runs Lab

This folder is the isolated review space for Stitchly workflow-run UI studies.

Purpose:

- define the visual language for workflow run lifecycle states
- separate run-state design from the main dashboard and canvas work
- review status surfaces before implementation in the production app

What belongs here:

- run list and run history studies
- run status chip and summary states
- run detail / inspection surfaces
- logs, events, and retry / cancellation affordances

Current convention:

- `v1/00_xxx/`
  Ordered run-state and run-surface studies for the first pass.

Current versions:

- `v1/`
  First pass for workflow run lifecycle design.
