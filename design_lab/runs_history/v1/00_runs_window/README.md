# 00 Runs Window

Purpose:

- explore a first canvas-popup run-history window
- merge the compact analytical table feel from the reference with the current Stitchly left-rail popup language
- test whether KPI chips, filters, and a dense table can coexist in the same popup without feeling overpacked

Included in this study:

- left menu rail with an active `Runs` item
- popup window anchored to the rail
- `Runs & Logs` header and export action
- filter chips and search control
- summary KPI row
- dense run-history table with status, error, and retries columns

What to review:

- whether the popup width feels right next to the rail
- whether the KPI row is useful or too heavy
- how dense the table can be before readability drops
- whether the lighter analytical structure should stay slightly brighter than the rest of the canvas chrome
