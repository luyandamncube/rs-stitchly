Retrying Direction

Retrying is active, but it also carries failure memory. The node should feel live and unstable, not as clean as `running`.

Active with tension Keep the node energized, but a little rougher than normal running.
Footer proves the retry The attempt counter helps distinguish retry from ordinary execution.
Not as terminal as failed The node still has a path back to success, so avoid full error weight.
