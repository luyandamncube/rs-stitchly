# 05 Venture

This folder holds the first docs-first venture framework for evaluating what Stitchly should help build next.

The goal is to make new ideas comparable before implementation starts, especially when those ideas could become:

- public utility products
- SaaS tools
- business-OS modules
- data-heavy operator products with Stitchly as the backbone

This framework is intentionally biased toward:

- fast first-dollar validation
- practical recurring-income potential
- manageable delivery cost
- ideas where Stitchly creates structural leverage rather than incidental reuse

## Contents

- `00_idea_framework.md`
  Defines the operating loop for how ideas move from raw capture to scored recommendation.

- `01_idea_record_template.md`
  Defines the fixed schema every idea record should use.

- `02_scoring_rubric.md`
  Defines the weighted rubric, scoring math, and confidence handling.

- `03_recommendation_rules.md`
  Defines deterministic recommendation outputs and gating rules.

- `04_portfolio_backlog.md`
  Tracks the current scored portfolio and next actions.

- `20_idea_savr.md`
  Seeded evaluation for Savr.

- `21_idea_waterwatch.md`
  Seeded evaluation for WaterWatch.

- `22_idea_gridza.md`
  Seeded evaluation for GridZA.

## Working Rule

Every new idea should:

1. start as a record using `01_idea_record_template.md`
2. be scored using `02_scoring_rubric.md`
3. receive a deterministic recommendation from `03_recommendation_rules.md`
4. be added or updated in `04_portfolio_backlog.md`

This keeps the framework useful as an operating system rather than a loose brainstorming folder.
